# Internationalization (i18n) Guide

This document explains how the Events Platform handles multilingual support and how to add new languages or translation keys.

## Architecture

The frontend uses [vue-i18n](https://vue-i18n.intlify.dev/) v11 in **Composition API mode** (`legacy: false`). All translations are statically defined in TypeScript locale files — no remote loading or lazy-loading of messages is needed for the current set of supported languages.

### Key files

| File | Purpose |
|------|---------|
| `src/i18n/index.ts` | i18n plugin creation, locale detection, persistence helpers |
| `src/i18n/locales/en.ts` | English translations (default / fallback) |
| `src/i18n/locales/sk.ts` | Slovak translations |
| `src/i18n/locales/de.ts` | German translations |
| `src/components/layout/LanguageSwitcher.vue` | Language dropdown in the header |

### Locale detection priority

1. **localStorage** (`app_locale` key) — persisted user choice
2. **`navigator.languages`** — browser preference
3. **Fallback** — `en`

### Fallback behavior

When a translation key is missing in the active locale, vue-i18n automatically falls back to the English (`en`) message. In production (`import.meta.env.DEV === false`), missing-key warnings are suppressed so users never see raw placeholder keys.

## How to add a new language

### 1. Create the locale file

Copy `src/i18n/locales/en.ts` to `src/i18n/locales/<code>.ts` (e.g. `it.ts` for Italian) and translate all values.

```bash
cp src/i18n/locales/en.ts src/i18n/locales/it.ts
```

### 2. Register the locale

In `src/i18n/index.ts`:

```ts
import it from './locales/it'

export type SupportedLocale = 'en' | 'sk' | 'de' | 'it'
export const SUPPORTED_LOCALES: SupportedLocale[] = ['en', 'sk', 'de', 'it']

// In the createI18n call, add to messages:
messages: {
  en,
  sk,
  de,
  it,
},
```

### 3. Add the language name entry

In **every** locale file (en, sk, de, and the new file), add an entry in the `languages` section:

```ts
languages: {
  en: 'English',
  sk: 'Slovenčina',
  de: 'Deutsch',
  it: 'Italiano',   // always in the native language
},
```

### 4. Run the tests

```bash
npm run test:unit
```

The i18n unit tests in `src/i18n/__tests__/i18n.test.ts` automatically verify that all locales have the same top-level and nested keys as English, and that no value is an empty string.

## How to add a new translatable key

### 1. Add the key to `en.ts`

Add the key in the appropriate section (e.g. `home`, `filters`, `eventDetail`):

```ts
// src/i18n/locales/en.ts
home: {
  // ...existing keys...
  newFeatureMessage: 'This is a new feature!',
},
```

### 2. Add translations to all other locale files

Add the same key to `sk.ts` and `de.ts` with translated values.

### 3. Use the key in a Vue component

```vue
<script setup lang="ts">
import { useI18n } from 'vue-i18n'
const { t } = useI18n()
</script>

<template>
  <p>{{ t('home.newFeatureMessage') }}</p>
</template>
```

### 4. For interpolation (dynamic values)

```ts
// In locale file:
greeting: 'Hello, {name}!'

// In template:
{{ t('home.greeting', { name: userName }) }}
```

## Locale-aware date/time/number formatting

Instead of hardcoding `'en-US'` in `toLocaleDateString()` / `toLocaleTimeString()`, use the `locale` ref from `useI18n()`:

```ts
const { locale } = useI18n()

function formatDate(dateStr: string): string {
  return new Date(dateStr).toLocaleDateString(locale.value, {
    month: 'short',
    day: 'numeric',
    year: 'numeric',
  })
}
```

This ensures dates, times, and numbers respect the user's chosen language.

## Special characters in translation strings

- **`@` symbol**: vue-i18n treats `@` as a special character in message syntax. Escape it as `{'@'}` in translation strings, and use double quotes in the TS file:
  ```ts
  emailPlaceholder: "your{'@'}email.com",
  ```
- **Apostrophes**: Use straight ASCII apostrophe `'` (U+0027), never curly quotes `'` (U+2019). This ensures Playwright assertions match the rendered text.

## Testing

### Unit tests
- `src/i18n/__tests__/i18n.test.ts` — tests locale detection, persistence, and key completeness
- Run with: `npm run test:unit`

### E2E tests
- `e2e/i18n.spec.ts` — tests language switching, persistence, and localized UI in all supported languages
- Run with: `npm run test:e2e`
